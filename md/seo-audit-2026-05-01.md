# DiscoverCloudcroft.com — Technical SEO Audit

Audit date: 2026-05-01
Repo: `/Users/chrishea/Library/Mobile Documents/com~apple~CloudDocs/GitHub/discover-beta`
Files reviewed: 108 HTML files (root + section dirs), `css/common.css`, `css/fonts.css`, `js/common.js`, `robots.txt`, `sitemap.xml`. Worktree copies under `.claude/worktrees/` were excluded.

---

## A. Executive summary

The site has a solid editorial foundation and a deliberate, schema-first page template. Most pages have JSON-LD, OG tags, a canonical, an `<html lang="en">`, a viewport meta, an h1, IBM Plex via `font-display: swap`, and a single shared CSS/JS pair. Word counts are healthy (only 5 pages under 300 words). The shelf/ directory's two stub pages already carry `noindex`. None of the JSON-LD blocks fail to parse.

That said, several issues today actively block indexing or send Google the wrong signal:

1. **40 of 105 canonicals point to a URL that is not the page's own filename** (e.g. `do/golf.html` canonicals to `/do/where-to-play-golf-in-cloudcroft-2026.html`, which does not exist on disk). Google will treat the named canonical as authoritative and may drop the real page.
2. **46 pages reference a missing OG image** (`media/og-default.webp` does not exist on disk). Social previews and the image asset's role in rich results are silently broken.
3. **Six "stub" pages should not be indexed but are**: the two root-level `complete-guide-...copy.html`, `index copy.html`, `do/...copy.html`, `stay/xxx-...copy.html`, and `topic-page-target-mockup.html`. None carry `noindex`. Three of them duplicate the homepage's exact title and meta description.
4. **167 internal `<a href>` links resolve to files that do not exist** (e.g. `stay/lodging.html` is referenced 28 times across the site).
5. Title length is consistently over Google's 60-char SERP cutoff (median 85, max 150).

`robots.txt` is permissive and correct, the XML sitemap is hand-maintained but lists one URL that doesn't exist on disk and omits 3 real public pages. There is no `Organization`, `WebSite`, or `SearchAction` schema on the homepage. Image dimensions are missing on essentially every `<img>` (469 of 486), creating CLS risk. 100 of 108 pages have a hero block but no `<link rel="preload" as="image">` for the LCP image. `js/common.js` is loaded as a render-blocking script on 84 pages and `defer`-ed on only 14.

**Overall site health: 58/100.** The architecture is right. The metadata layer is leaking value.

## C. High-priority issues (P1)

### C1. Title length is over the SERP cutoff on 87 of 106 pages
- Severity: **P1**
- Median 85, max 150 chars. Google generally truncates at ~60.
- Files affected (top 5 of 87):
  - `shop/elk-shed.html:7` — 150 chars
  - `shop/burro-ave-trading-post.html:7` — 140 chars
  - `shop/coco-blu.html` — 134 chars
  - `eat/big-daddys.html` — 134 chars
  - `eat/brewery.html` — 129 chars
- Pattern: most titles are `[Entity], Cloudcroft NM (2026) — [tagline] | DiscoverCloudcroft.com`. The em-dashed tagline + `| DiscoverCloudcroft.com` always pushes past 60.
- Fix: Drop ` | DiscoverCloudcroft.com` (Google often appends it as the site name automatically when configured via Schema). Trim taglines. Aim for 50–60 chars including the geo qualifier.
- Impact: **High** — affects almost every page's CTR.

### C2. Meta descriptions over 160 chars on 83 pages, over 200 chars on 44
- Severity: **P1**
- Median 182. Max 313.
- Files affected (longest):
  - `stay/cabins-at-cloudcroft.html` — 267
  - `stay/dusty-boots-motel-cloudcroft.html` — 249

  - `stay/osha-trail.html` — 233
  - `stay/stay-the-summit-inn-Cloudcroft-2026.html` — 228
- Why it hurts: Google rewrites long descriptions, often badly. Lost control of SERP messaging.
- Fix: Tighten to 140–155 chars with one CTA verb + one differentiator + geo. Worth running a script across all `<meta name="description">` content="…".
- Impact: **Medium-High**.

### C3. Homepage is missing `Organization`, `WebSite`, and `SearchAction` schema
- Severity: **P1**
- File: `index.html`
- Currently has: `TouristDestination`, `BreadcrumbList`, `FAQPage`, and 8 `Event` items in an `@graph`. No `Organization` (publisher) or `WebSite` (with `potentialAction: SearchAction`).
- Why it hurts: Loses sitelinks-search-box eligibility. Loses the publisher/`sameAs` social-graph attachment that strengthens E-E-A-T.
- Fix: Add a fourth JSON-LD block on `index.html` with `Organization` (logo + `sameAs` to all social profiles + `contactPoint`) and `WebSite` (with `potentialAction` of type `SearchAction`).
- Impact: **High** for brand SERP appearance.

### C4. 469 of 486 `<img>` tags omit `width` and `height`
- Severity: **P1**
- This causes layout shift (CLS) when images load. 25 of those also omit `loading=` entirely; the rest at least have `loading="lazy"`.
- Files affected (representative): `index.html` has 50+ images without dimensions; entity pages average 4-6 images each, all without dimensions. Examples: `index.html` images including `media/image-do-pickleball-group.webp`, `do/do-photo/hike/hike-00457.jpg`, `media/mexican-canyon-trestle-bridge.jpeg`.
- Fix: Add `width` and `height` to every `<img>`. Since this is a static site, write a small Python helper that reads each referenced image with `PIL` and rewrites the `<img>` tag with intrinsic dimensions. Alternatively, set CSS `aspect-ratio` on the wrapper.
- Impact: **High** — Core Web Vitals "CLS" failure is currently very likely.

### C5. Hero image not preloaded on 100 of 108 pages
- Severity: **P1**
- Only `index.html`, `complete-guide-to-cloudcroft-new-mexico-2026 copy.html`, and a few others use `<link rel="preload" as="image">`.
- Most templates use a CSS `background-image` on `.hero-bg`, which the browser cannot pre-fetch until the CSS parses.
- Fix: Each template should preload the LCP image:
  ```html
  <link rel="preload" as="image" href="...path-to-hero..." fetchpriority="high">
  ```
  Also add `fetchpriority="high"` on the hero `<img>` if you switch from CSS-bg to `<img>`.
- Impact: **High** for LCP.

## H. Long-term roadmap

1. **Replace hand-maintained `sitemap.xml` and `content/sync-nav.py` with a single build step** that walks the file tree, computes nav prefixes, and emits both. Eliminates the entire class of nav drift / sitemap drift bugs.
2. **Adopt a static site generator (Eleventy, Astro, or Hugo)** without changing the visual output. The project is large enough (108 pages, 8 sections, near-identical templates) that the discipline of templating will eliminate ~60% of the issues in this report. The "no-build" stance is what made this audit necessary.
3. **Add HTML validation, link-checking, and Lighthouse CI to a GitHub Action.** Block merges that increase HTML errors or drop CWV scores.
4. **Image pipeline**: standardize on `.webp` at 1200/800/400 widths; emit `<picture>` with `srcset` automatically. Drop HEIC and `.tif` from `media/`.

9. **Schema.org coverage expansion**: `MenuItem`, `Menu` for restaurant pages; `LodgingReservation` (where bookable); `TrailEntity` (custom) or `TouristAttraction` for hikes. Use Google's Rich Results Test against every published page in CI.
