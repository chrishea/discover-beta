DEPLOY ORDER MATTERS — DO THIS FIRST
================================================================================

The redirects must NOT be live before the renamed pages exist on Amplify.
Otherwise users get 301 → 404.

Correct sequence:

  Step 1.  Push (or manually deploy) the repo with the renamed files to
           Amplify FIRST. Wait for the build/deploy to finish (Amplify
           console shows green "Deployed" badge on the latest commit).

  Step 2.  THEN add the redirects in step 4 above. Test with curl.

If your Amplify app is connected to a git branch (most common): just
push to that branch and Amplify auto-deploys. If it's a manual deploy
(upload zip): zip the repo root excluding .git, .claude, drafts/, shelf/,
media-originals/, and node_modules — then "Deploy without Git provider"
in the Amplify console.
